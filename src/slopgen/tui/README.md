# tui

**Form kit (`forms.py`):** screen controls are *declared*, not hand-wired. A `Form` holds a list of `Field`s (`Text` / `Number` / `Choice` / `Toggle` / `Heading` / `Note`, plus `Group` for a gated sub-section) and can build its widgets (`compose`/`build`), read every value back typed (`read` → dict), prefill from a dict (`fill`), and toggle each `Group` from a predicate over the current values (`refresh_visibility`, which also scrolls a freshly-revealed group into view). `Text` is the single text-field object: regular text fields reject newlines and Enter moves focus like Tab, while `large=True` fields always show 5 rows and scroll after that; `Number` fields use the same value plumbing plus `↑`/`↓` buttons for ±1, and password fields keep `Input` internally for secret entry. Widget ids are `{form.ns}-{field.key}`. Both the wizard panes and every config-entity form are built from field lists this way — editing the UI means editing a list. (Layout note: the `ContentSwitcher` bodies are pinned to `height: 1fr` and `.pane` to `height: 100%` so the inner `VerticalScroll` actually scrolls instead of overflowing off-screen — the earlier `height: auto` collapse is why panes only appeared when the window was stretched.)

**EN:** Textual TUI with a custom Minecraft theme and no footer — a custom `TopBar` (title, RU/EN toggle, `<-` back, Palette) docks on top of every screen; all labels go through the `I18N` table; the chosen theme persists to `[ui].theme`. Screens: `HomeScreen` (centered arrow-key menu of big buttons), `GenerateScreen` (5-step wizard: content / visuals with profile prefill + full overrides / ads incl. fully manual / publish / summary with a live CLI-command preview), `ProgressScreen` (queue table + live log, pipeline in a worker thread), `ConfigScreen` (left sections; `EntityPane` gives each entity kind a top tab-button row per config file plus "+ new", prefilled forms, save and confirmed delete; `LLMPane` adds provider/model presets, a web-search toggle, a password API-key input persisted to `.env` via `set_env_var`, and ★ profile activation; `FootagePane` saves the Pexels/Pixabay keys to `.env`).

**RU:** TUI на Textual с темой Minecraft и без футера — на каждом экране свой `TopBar` (заголовок, RU/EN, `<-`, Palette); все надписи через таблицу `I18N`; выбранная тема сохраняется в `[ui].theme`. Экраны: `HomeScreen` (центрированное меню крупных кнопок со стрелочками), `GenerateScreen` (визард из 5 шагов: контент / видеоряд с предзаполнением профиля и полными переопределениями / реклама вплоть до ручной / публикация / итог с живым превью CLI-команды), `ProgressScreen` (таблица очереди + лог, пайплайн в worker-потоке), `ConfigScreen` (секции слева; `EntityPane` даёт каждому виду сущностей ряд таб-кнопок по конфиг-файлам плюс «+ новый», предзаполненные формы, сохранение и удаление с подтверждением; `LLMPane` добавляет пресеты провайдеров/моделей, тумблер веб-поиска, password-ввод API-ключа с сохранением в `.env` через `set_env_var` и активацию профиля ★; `FootagePane` сохраняет ключи Pexels/Pixabay в `.env`).
