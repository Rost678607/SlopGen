# pipeline

**EN:** The conveyor. `job.py` — `VideoJob`/`Scene`/`BgAsset`/`FgInsert`/`Word` state passed between stages. `context.py` — `AppContext` with resolved configs (incl. the visuals profile and the active LLM profile), the LLM client and topic history. `orchestrator.py` — runs the stage chain per video, reports progress via an `on_event` callback (consumed by both CLI and TUI); one failed video doesn't kill the batch. Stages in `stages/`: idea (skipped when the user supplied `--idea`) → script (scaled to the target duration, emits per-beat visual queries) → tts → footage (fills background assets + foreground inserts per the visuals profile) → subtitles → assemble → metadata.

**RU:** Конвейер. `job.py` — состояние `VideoJob`/`Scene`/`BgAsset`/`FgInsert`/`Word` между стадиями. `context.py` — `AppContext` с резолвнутыми конфигами (включая профиль видеоряда и активный LLM-профиль), LLM-клиентом и историей тем. `orchestrator.py` — гонит цепочку стадий по каждому ролику, шлёт прогресс через `on_event` (слушают CLI и TUI); падение одного ролика не убивает батч. Стадии в `stages/`: idea (пропускается при `--idea`) → script (масштабируется под целевую длительность, выдаёт визуальные биты) → tts → footage (заполняет фон и вставки по профилю видеоряда) → subtitles → assemble → metadata.
