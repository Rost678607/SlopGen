# llm

**EN:** `ChatLLM` — one client for any OpenAI-compatible provider (JSON mode); provider presets in `PROVIDERS` (deepseek, gemini, openrouter, custom) and popular model choices in `MODEL_PRESETS`. Connections are described by `LLMProfile` configs (`configs/llm/*.toml`), the active one selected by `[llm].profile`; empty fields fall back to provider defaults via `resolve_provider`. API keys live in `.env` only. When a profile has `web_search`, `complete_json` runs a tool-calling loop (`_run_tools`) offering the real `web_search` tool from `tools.py` (keyless DuckDuckGo) — the model calls it, results are fed back, then it answers.

**RU:** `ChatLLM` — один клиент для любого OpenAI-совместимого провайдера (JSON-режим); пресеты провайдеров в `PROVIDERS` (deepseek, gemini, openrouter, custom), популярные модели — в `MODEL_PRESETS`. Подключения описываются профилями `LLMProfile` (`configs/llm/*.toml`), активный выбирается `[llm].profile`; пустые поля подменяются дефолтами провайдера через `resolve_provider`. Ключи API — только в `.env`. Если у профиля включён `web_search`, `complete_json` запускает цикл tool-calling (`_run_tools`) и предлагает настоящий инструмент `web_search` из `tools.py` (бесключевой DuckDuckGo) — модель его вызывает, результаты возвращаются, затем она отвечает.
